Menubutton - Boutons de menu¶

Un bouton de menu, Menubutton, est la partie visible d’un menu déroulant. Un tel bouton est donc associé à un widget Menu (voir plus loin) qui sert à contenir (et à afficher) les choix qui apparaîssent lorsque l’utilisateur clique sur le bouton.

Pour créer un bouton de menu comme enfant d’une fenêtre ou d’un cadre parent:

class Menubutton(parent, option, ...)¶

Le constructeur retourne le nouveau bouton de menu. Ses options sont:

Paramètres:
  • activebackground – La couleur de fond utilisée lorsque la souris survole le widget. Voir Les couleurs.
  • activeforeground – La couleur d’avant plan (texte) utilisée lorsque la souris survole le widget.
  • anchor – Cette option sert à préciser la position du texte du bouton si celui-ci dispose de plus de place que de besoin pour le texte. Sa valeur par défaut est 'center' ce qui centre le texte sur le bouton. Pour d’autres valeur possibles, voir Le système d’ancrage. Par exemple, si anchor='w', le texte sera centré verticalement contre le bord gauche du bouton.
  • bg – (ou background) La couleur de fond utilisée lorsque la souris ne survole pas le bouton.
  • bitmap – Pour afficher un bitmap sur le bouton de menu, configurez cette option avec le nom de ce bitmap; voir Les bitmaps.
  • bd – (ou borderwidth) Épaisseur de la bordure dessinée autour du bouton de menu. 2 pixels par défaut. Pour les valeurs possibles, voir Les dimensions.
  • compound – Si vous utilisez à la fois du texte et un graphique (soit un bitmap soit une image), cette option sert à indiquer où le graphique apparaît par rapport au texte. Les valeurs possibles sont 'none' (par défaut), 'top', 'bottom', 'left', 'right' et 'center'. Par exemple, compound='right' positionnera le graphique à la droite du texte. Si vous conservez compound='none', le graphique sera affiché mais pas le texte.
  • cursor – Le pointeur de souris utilisé lorsqu’elle survole ce widget. Voir Le pointeur de la souris.
  • direction – Normalement, le menu apparaît en dessous du bouton. Utilisez direction='left' pour afficher le menu sur le côté gauche du bouton, direction='right' pour l’afficher à sa droite; direction='above' pour l’afficher au-dessus.
  • disabledforeground – La couleur d’avant plan (texte) utilisée lorsque le bouton de menu est désactivé.
  • fg – (ou foreground) La couleur d’avant plan utilisée lorsque la souris ne survole pas le bouton.
  • font – Sert à préciser la police de caractères utilisée pour afficher le texte. Voir Les polices de caractères.
  • height – La hauteur du bouton de menu exprimée en lignes de texte (non en pixels !). Par défaut, le bouton s’ajuste à son contenu.
  • highlightbackground – La couleur de la ligne de focus lorsque le widget n’a pas le focus. Voir Le focus: réception des saisies clavier.
  • highlightcolor – Couleur de la ligne de focus lorsque le widget a le focus.
  • highlightthickness – Épaisseur de la ligne de focus.
  • image – Pour afficher une image sur un bouton de menu, passer la référence de l’image à cette option. Voir Les images.
  • justify – Cette option sert à indiquer l’alignement du texte s’il contient plusieurs lignes: justify='left' pour un alignement à gauche (valeur par défaut); justify='center' pour centrer, ou justify='right' pour un alignement à droite.
  • menu – Pour associer un ensemble de choix au bouton de menu, configurer cette option avec un widget Menu qui contient ces choix. Ce widget Menu doit être un enfant du bouton: il doit être créé en utilisant le bouton de menu comme premier argument de son constructeur. Voir plus loin pour un exemple qui montre comment associer un bouton de menu avec un menu.
  • padx – Sert à indiquer un espace horizontal à mettre de chaque côté (gauche et droit) du bouton. 1 pixel par défaut.
  • pady – Similaire à padx mais dans la direction verticale. 1 pixel par défaut.
  • relief – Sert à préciser le relief à utiliser pour dessiner les contours du bouton; 'raised' par défaut. Pour d’autres effets, voir Les styles de relief.
  • state – Normalement, un bouton de menu est réactif à la souris. Utilisez state='disabled' pour le griser et le rendre inactif.
  • takefocus – Normalement, les boutons de menu n’obtiennent pas le focus lorsque l’utilisateur appuie sur la touche Tab (voir Le focus: réception des saisies clavier) Utilisez takefocus=True pour qu’il puisse obtenir le focus comme cela.
  • text – Pour afficher un texte sur le bouton de menu, configurez cette option avec le texte voulu, donné sous la forme d’une chaîne de caractères. Utiliser le caractère spécial '\n' pour faire des sauts de ligne.
  • textvariable – Vous pouvez associer une variable de contrôle StringVar à ce bouton de menu par l’intermédiaire de cette option. Toute modification de sa valeur est répercutée sur le bouton et vice versa. Voir Les variables de contrôle: Les valeurs sous les widgets.
  • underline – Sert à souligner l’un des caractères du texte en précisant sa position (index) dans le texte. Par défaut, aucun caractère n’est souligné.
  • width – Largeur du bouton de menu en nombre de caractères (pas en pixels!). Si aucune valeur n’est précisée, le bouton s’ajuste à son contenu.
  • wraplength – Normalement, les lignes ne sont pas «enveloppées». Vous pouvez configurer cette option avec un nombre de caractères et toutes les lignes seront coupées de façon à ne pas avoir plus de caractères que le nombre indiqué.

Voici un court exemple qui illustre la création d’un bouton de menu et du menu associé lequel est composé de deux cases à cocher:

mb = Menubutton(root, text='condiments',
                         relief='raised')
mb.grid()

mb.menu = Menu(mb, tearoff=0)
mb['menu'] = mb.menu

mayoVar  = IntVar()
ketchVar = IntVar()
mb.menu.add_checkbutton(label='mayo',
    variable=mayoVar)
mb.menu.add_checkbutton(label='ketchup',
    variable=ketchVar)

Dans cet exemple, on crée un bouton de menu étiqueté “condiments”. Lorsque qu’on clique dessus, deux cases à cocher étiquetées “mayo” et “ketchup” sont affichées en dessous du bouton.
