Présentation du site io.adafruit.com

Commencez par créer un compte gratuit sur io.adafruit.com. Une fois connecté, cliquez sur Feeds pour voir/créer de nouveaux topic.

Créez un nouveau Feed. Ici, nous utiliserons le Feed nommé test. Vous voyez au passage qu'il a déjà reçu des informations : la dernière valeur reçue - 18.7 - correspond à la température envoyée par notre objet connecté.

Fondamental : Récupérer la clé API

Sur le menu de gauche, vous trouverez un lien View AIO Key. Cliquez dessus et notez la clé IO_KEY qui vous servira a vous authentifier sur le service. Sans cette clé, impossible de publier une valeur dans votre Feed. Cette clé est une chaîne de 32 chiffres et lettres.

Complément : Tableaux de bord

Vous pouvez présenter les données remontées d'un Feed dans un tableau de bord (dashboard). Un dashboard contient plusieurs blocs qui peuvent avoir des apparences diverses : jauge, bouton, texte, graphique, curseur etc... Choisissez le type de bloc le plus adapté aux données que vous voulez présenter.

Ici, notre Feed est une température. On peut le présenter au moyen d'un bloc de type jauge. Voici ce que cela donne.

L'idée est de recueillir les données de plusieurs capteurs et de les présenter de manière graphique sur un tableau de bord, par exemple les températures dans toutes les pièces de la maison.

Attention :

Dans sa version gratuite, le site io.adafruit.com limite à

  • 10 le nombre de Feeds que l'on peut recevoir

  • 5 le nombre de Dashboards

  • 30 envois de données par minute

  • 30 jours d'historique sur les données

Si vous avez besoin de plus, vous pouvez souscrire un abonnement ou installer sur raspberry Pi une suite logicielle équivalente. Je vous conseille node-red + mosquitto
